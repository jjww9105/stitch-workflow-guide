S Stitch Workflow
Steps Prompts Deploy Cheatsheet

Codex + Google Stitch + Reference Design

제품 의미와 디자인 감성을 분리해서 Stitch에 넘기는 실전 워크플로우

Codex는 제품을 해석하고, Stitch는 새 UI를 만들고, 레퍼런스 이미지는 DESIGN.md/design system을 만드는 데만 사용한다.

프롬프트 바로 쓰기 GitHub Pages 배포 안내
제일 중요한 규칙

기존 Codex UI는 디자인 참고자료가 아니다. 제품 목적, 기능, 화면 의도, 데이터 상태만 추출한다.

레퍼런스 이미지는?

색감, 폰트 느낌, 여백감, 컴포넌트 분위기만 추출한다. 원본 레이아웃과 브랜드는 복사하지 않는다.

Overview

따라 할 순서

다음에 기억이 흐릿하면 이 순서대로만 진행하면 된다.

  1. 1

    제품 프로젝트를 Codex에서 연다

    새 프론트엔드를 만들 실제 프로젝트 폴더를 연다.

  2. 2

    Codex가 Semantic Rebuild Brief를 만든다

    아래 01 프롬프트를 제품 프로젝트 Codex에 붙인다.

  3. 3

    Stitch가 제품 UI 예시를 만든다

    Codex가 만든 Brief 전체와 02 프롬프트를 Stitch에 같이 붙인다.

  4. 4

    레퍼런스 이미지로 DESIGN.md를 만든다

    Stitch에 캡처 이미지를 올리고, 이미지를 선택한 뒤 03 프롬프트를 붙인다.

  5. 5

    제품 UI에 DESIGN.md를 적용한다

    제품 UI 예시와 DESIGN.md/design system을 선택하고 04 프롬프트를 붙인다.

  6. 6

    품질 검사 후 zip을 Codex에 통합한다

    05로 검사하고, 다운로드한 zip은 06 프롬프트로 제품 프로젝트에 통합한다.
