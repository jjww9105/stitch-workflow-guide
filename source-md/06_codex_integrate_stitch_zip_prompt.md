# 06. Stitch zip을 Codex 프로젝트에 통합할 때 쓰는 프롬프트

사용 위치: 실제 제품의 Codex 프로젝트 폴더

사용법:

1. Stitch에서 결과물을 zip으로 다운로드한다.
2. 실제 제품 프로젝트를 Codex에서 연다.
3. zip 파일 경로와 함께 아래 프롬프트를 붙여넣는다.

```text
Stitch에서 다운로드한 zip을 현재 프로젝트의 새 프론트엔드 구현 재료로 사용해서 통합해줘.

중요:
Stitch zip을 그대로 덮어쓰지 마.
현재 프로젝트의 실제 스택, 라우팅, 상태관리, 데이터 흐름, 빌드 설정을 먼저 확인해.
Stitch 결과물의 일부 예쁜 화면만 가져오지 말고, Semantic Rebuild Brief의 전체 화면/탭/라우트 커버리지를 기준으로 통합해.

해야 할 일:
1. 현재 프로젝트 구조와 프론트엔드 스택을 확인해.
2. 현재 프로젝트의 전체 탭/라우트/화면/기능 인벤토리를 확인해.
3. Stitch zip 내부 구조와 화면 목록을 확인해.
4. Brief의 전체 화면 인벤토리와 Stitch zip의 화면 세트를 비교해.
5. 누락된 화면, 누락된 상태, 누락된 라우트, 누락된 주요 액션을 목록화해.
6. 어떤 파일/컴포넌트/스타일을 가져올지 계획해.
7. 기존 기능, 라우팅, API 연동, 상태관리, 인증, 데이터 처리를 보존해.
8. Stitch 결과물의 디자인, 레이아웃, 컴포넌트 아이디어를 현재 프로젝트 구조에 맞게 통합해.
9. 하드코딩된 더미 데이터가 있으면 실제 데이터 흐름에 맞게 바꿔.
10. loading / empty / error / success / permission / validation 상태를 실제 코드에 반영해.
11. 접근성, 반응형, 텍스트 overflow를 확인해.
12. 빌드 또는 테스트를 실행해서 깨진 부분을 수정해.

주의:
- 기존 프로젝트 기능을 삭제하지 마.
- 사용자 변경사항을 되돌리지 마.
- Stitch 코드를 무비판적으로 복붙하지 마.
- Stitch zip의 일부 화면만 보고 전체 앱 통합이 끝났다고 판단하지 마.
- 기존 Codex UI의 낡은 시각 스타일로 되돌리지 마.
- 레퍼런스 사이트의 브랜드/문구/로고를 복사하지 마.

출력:
1. 전체 화면/탭/라우트 커버리지 비교
2. 통합 계획
3. 적용한 파일 목록
4. 유지한 기존 기능
5. 수정한 UI/컴포넌트
6. 추가하거나 보강한 누락 화면/상태
7. 실행한 검증 명령
8. 남은 리스크

Stitch zip 경로:
[여기에 zip 경로 입력]
```
