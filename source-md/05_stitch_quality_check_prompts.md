# 05. Stitch 품질 검사 프롬프트

사용 위치: Google Stitch

이 파일은 선택 사항이지만 강하게 추천한다.  
생성된 화면이 좋아 보여도 전체 화면 커버리지, 제품 의미, 상태, 디자인 시스템이 빠졌을 수 있다.

## 1. 화면 커버리지 검사

```text
선택한 Stitch 결과가 Semantic Rebuild Brief의 전체 화면/탭/라우트/상태 인벤토리를 빠짐없이 반영했는지 검사해줘.

언어 규칙:
- 모든 화면 문구, 버튼, 라벨, 안내문, 상태 메시지는 한국어로 유지하거나 수정해.

검사할 것:
- Brief에 있는 전체 탭이 모두 반영되었는지
- Brief에 있는 전체 라우트/페이지가 모두 반영되었는지
- 핵심 화면, 상세 화면, 모달, 드로어, 설정, 온보딩이 필요한 만큼 반영되었는지
- loading / empty / error / success / permission / validation 상태가 필요한 화면에 있는지
- 일부 화면만 예쁘게 만들고 전체 앱을 만든 것처럼 끝내지 않았는지

반환:
1. Brief 기준 전체 화면 인벤토리
2. 현재 Stitch 결과에 포함된 화면
3. 누락된 화면/탭/라우트/상태
4. 바로 추가하거나 다음 배치에서 만들어야 할 화면
5. 화면 커버리지 판단: 충분함 / 부족함
```

## 2. 기존 Codex UI 오염 검사

```text
선택한 전체 화면 세트에 기존 Codex UI의 시각적 흔적이 우연히 남아 있는지 검사해줘.

언어 규칙:
- 모든 화면 문구, 버튼, 라벨, 안내문, 상태 메시지는 한국어로 유지하거나 수정해.

기존 Codex UI가 영향을 주면 안 되는 것:
- 레이아웃
- 색상
- 타이포그래피
- 컴포넌트 구조
- 탭 배치 방식
- 여백
- 섹션 순서
- 정보 위계
- 장식 스타일

기존 Codex UI에서 가져와도 되는 것은 오직 다음뿐이야:
- 제품 목적
- 기능
- 탭/라우트/화면의 존재
- 사용자 흐름
- 화면 의도
- 데이터 상태
- 누락 상태
- 피해야 할 anti-pattern

기존 Codex UI의 시각적 흔적이 보이면, 제품 의미와 선택한 DESIGN.md는 유지하면서 그 흔적을 제거해줘.

반환:
1. 발견한 기존 UI 시각 흔적
2. 영향받은 화면
3. 수정한 내용
4. 남은 가정
```

## 3. 레퍼런스 사이트 과복사 검사

```text
선택한 전체 화면 세트가 레퍼런스 사이트를 너무 그대로 베꼈는지 검사해줘.

언어 규칙:
- 모든 사용자-facing 문구는 한국어로 유지하거나 수정해.

레퍼런스 이미지가 영향을 줘도 되는 것:
- mood
- 품질 기준
- 색감
- 타이포그래피 느낌
- 여백 리듬
- 컴포넌트 분위기
- 상태 표현 방식

레퍼런스 이미지에서 복사하면 안 되는 것:
- 정확한 레이아웃
- 섹션 구조
- 브랜드 정체성
- 로고
- 문구/콘텐츠
- 고유한 시각 정체성

레퍼런스를 너무 문자 그대로 복사한 부분이 있으면, 디자인 시스템 방향만 유지하고 독립적인 제품 UI로 수정해줘.

반환:
1. 과하게 복사된 요소
2. 영향받은 화면
3. 수정한 내용
4. 유지한 레퍼런스의 좋은 특성
```

## 4. 상태 보강

```text
선택한 전체 화면 세트에서 빠진 제품 상태가 있는지 검사해줘.

언어 규칙:
- 모든 상태 메시지, 안내문, 버튼 문구는 한국어로 작성해.

추가하거나 수정할 상태:
- loading 상태
- empty 상태
- error 상태
- validation 상태
- success / confirmation 상태
- permission / locked 상태

각 상태는 사용자가 다음을 이해할 수 있게 해야 해:
- 무슨 일이 일어났는지
- 다음에 무엇을 할 수 있는지
- 시스템이 아직 처리 중인지

선택한 DESIGN.md와 일관되게 유지해.

반환:
1. 화면별 누락 상태
2. 수정된 상태 화면
3. 보강한 누락 상태
4. 가정
```

## 5. DESIGN.md 준수 검사

```text
선택한 전체 화면 세트가 선택한 DESIGN.md / design system을 제대로 따르는지 검사해줘.

언어 규칙:
- 화면 문구와 상태 메시지는 한국어로 유지해.

검사할 것:
- color token
- 타이포그래피
- 여백
- radius
- border
- shadow / elevation
- 버튼
- 카드
- form
- table / data display
- navigation / tab
- modal / drawer
- hover / focus 상태
- loading / empty / error / success / permission / validation 상태

DESIGN.md와 충돌하는 부분이 있으면 전체 화면 세트 기준으로 수정해.

반환:
1. 수정한 내용
2. 영향받은 화면
3. 적용한 DESIGN.md 규칙
4. 모호하거나 빠진 DESIGN.md 규칙
```

## 6. 모바일/데스크톱 검사

```text
선택한 전체 화면 세트가 모바일과 데스크톱에서 모두 실제로 쓰기 좋은지 검사해줘.

언어 규칙:
- 모바일/데스크톱 화면의 모든 사용자-facing 문구는 한국어로 유지해.

검사할 것:
- 주요 액션이 바로 보이는지
- 텍스트 크기가 읽기 좋은지
- 터치 영역이 충분한지
- 내비게이션과 탭 구조가 작은 화면에서도 자연스러운지
- 작은 화면에서 table/list가 깨지지 않는지
- form을 쓰기 쉬운지
- 모달/드로어가 모바일에서 과하게 답답하지 않은지
- 스크롤 동작이 자연스러운지
- 텍스트나 컨트롤이 겹치지 않는지

모바일과 데스크톱 모두에서 사용할 수 있게 수정해.

반환:
1. 모바일 수정 내용
2. 데스크톱 수정 내용
3. 화면별 반응형 리스크
4. 남은 반응형 가정
```
