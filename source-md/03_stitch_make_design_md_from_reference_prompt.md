# 03. Stitch에서 레퍼런스 이미지로 DESIGN.md 만들기

사용 위치: Google Stitch

목적:

레퍼런스 사이트의 화면을 복사하지 않고, 전체 화면 세트에 적용할 디자인 시스템만 추출한다.

사용법:

1. 원하는 레퍼런스 사이트 캡처 이미지를 Stitch에 올린다.
2. Stitch에서 그 이미지를 클릭하거나 선택한다.
3. 아래 프롬프트를 붙여넣는다.
4. 제품 유형은 실제 프로젝트에 맞게 바꾼다.

```text
선택한 레퍼런스 이미지를 바탕으로 DESIGN.md / design system을 만들어줘.

언어 규칙:
- DESIGN.md의 설명, do/don't, 사용 가이드, 예시 UI 문구는 한국어로 작성해.
- 디자인 토큰 이름은 `primary`, `surface`, `text-muted`처럼 영어 키를 써도 돼.
- 이 디자인 시스템을 적용해 생성될 화면의 사용자-facing 문구는 한국어 기준으로 설계해.

중요:
이 레퍼런스 이미지는 전체 화면 세트에 적용할 공통 디자인 시스템을 추출하기 위한 참고자료야.
원본 사이트의 레이아웃, 섹션 구조, 브랜드명, 로고, 문구, 콘텐츠, 고유한 정체성은 복사하지 마.
기능 구조나 제품 구조도 레퍼런스 사이트에서 가져오지 마.

추출하고 재해석할 것:
- 색상 팔레트
- semantic color token
- 타이포그래피 스타일
- 제목, 본문, 라벨, 데이터 텍스트 규칙
- 여백 리듬
- radius, border, shadow, elevation 규칙
- 버튼 스타일
- 카드 스타일
- form/input 스타일
- table / data display 스타일
- navigation / tab 스타일
- modal / drawer 스타일
- prompt/code block처럼 긴 텍스트를 다루는 UI 규칙
- hover, focus, loading, empty, error, success, permission, validation 상태
- 전체 mood
- do / don't 규칙

기능 계약 관련 규칙:
- DESIGN.md는 기능을 추가하거나 삭제하는 문서가 아니다.
- DESIGN.md는 화면의 색, 폰트, 여백, 컴포넌트, 상태 표현 규칙만 정의한다.
- Functional Contract의 액션, 데이터 흐름, validation, permission, error handling을 바꾸지 않는다.

디자인 시스템 목표:
Step 02에서 만든 전체 화면 세트의 모든 화면에 일관되게 적용할 수 있는 재사용 가능한 DESIGN.md를 만들어줘.
한 화면만 예쁘게 바꾸는 스타일 가이드가 아니라, 앱 전체에 적용되는 시스템이어야 한다.

대상 제품 유형:
[여기에 제품 유형 입력: 예: 업무용 SaaS 대시보드 / 데이터 관리 툴 / 소비자 앱 / 랜딩페이지 / 에디터 / 커머스 / 학습 도구 / 실행 가이드]

레퍼런스의 분위기를 이 제품 유형과 전체 화면 세트에 맞게 재해석해.

너무 문자 그대로 복제하지 마.
분위기와 품질 기준은 유지하되, 새로운 제품에 적용 가능한 독립적인 디자인 시스템으로 만들어줘.

출력:
1. DESIGN.md
2. 전체 화면 세트에 적용할 디자인 토큰과 컴포넌트 규칙
3. 상태 화면 규칙
4. 긴 텍스트/프롬프트/데이터 화면 규칙
5. 추출한 시각 방향에 대한 짧은 설명
6. 레퍼런스 이미지에서 의도적으로 복사하지 않은 것
```
