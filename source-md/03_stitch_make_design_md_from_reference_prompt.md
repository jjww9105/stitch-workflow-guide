# 03. 레퍼런스 이미지에서 DESIGN.md를 만들 때 쓰는 프롬프트

사용 위치: Google Stitch

사용법:

1. 원하는 레퍼런스 사이트 캡처 이미지를 Stitch에 올린다.
2. Stitch에서 그 이미지를 클릭하거나 선택한다.
3. 아래 프롬프트를 붙여넣는다.

```text
선택한 레퍼런스 이미지를 바탕으로 DESIGN.md / design system을 만들어줘.

언어 규칙:
- DESIGN.md의 설명, do/don't, 사용 가이드, 예시 UI 문구는 한국어로 작성해.
- 디자인 토큰 이름은 `primary`, `surface`, `text-muted`처럼 영어 키를 써도 돼.
- 이 디자인 시스템을 적용해 생성될 화면의 사용자-facing 문구는 한국어 기준으로 설계해.

중요:
이 레퍼런스 이미지는 시각적 분위기와 디자인 시스템 규칙을 추출하기 위한 참고자료야.
원본 사이트의 레이아웃, 섹션 구조, 브랜드명, 로고, 문구, 콘텐츠, 고유한 정체성은 복사하지 마.

추출하고 재해석할 것:
- 색상 팔레트
- semantic color token
- 타이포그래피 스타일
- 제목, 본문, 라벨, 데이터 텍스트 규칙
- 여백 리듬
- radius, border, shadow, elevation 규칙
- 버튼 스타일
- 카드 스타일
- form/input 스타일
- 필요하면 table / data display 스타일
- 내비게이션 스타일
- hover, focus, loading, empty, error, success 상태
- 전체 mood
- do / don't 규칙

디자인 시스템 목표:
다른 제품 UI에 적용할 수 있는 재사용 가능한 DESIGN.md를 만들어줘.

대상 제품 유형:
[여기에 제품 유형 입력: 예: 업무용 SaaS 대시보드 / 데이터 관리 툴 / 소비자 앱 / 랜딩페이지 / 에디터 / 커머스]

레퍼런스의 분위기를 이 제품 유형에 맞게 재해석해.

너무 문자 그대로 복제하지 마.
분위기와 품질 기준은 유지하되, 새로운 제품에 적용 가능한 독립적인 디자인 시스템으로 만들어줘.

출력:
1. DESIGN.md
2. 추출한 시각 방향에 대한 짧은 설명
3. 레퍼런스 이미지에서 의도적으로 복사하지 않은 것
```

## 제품 유형 예시

위 프롬프트의 `[여기에 제품 유형 입력]`에는 이런 식으로 적으면 된다.

```text
업무용 SaaS 대시보드
```

```text
반복 사용이 많은 데이터 관리 툴
```

```text
소비자용 모바일 웹 앱
```

```text
제품 소개용 랜딩페이지
```
